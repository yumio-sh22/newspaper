# Newspaper

## 1. Описание проекта

Данный проект представляет собой клиент-серверное веб-приложение, реализующее функциональность интернет-издания ("Газета"). Система позволяет пользователям создавать, редактировать и просматривать статьи, а администраторам и редакторам — управлять контентом и пользователями.

Приложение разработано в рамках курсовой работы и демонстрирует навыки работы с архитектурой клиент-сервер, реляционными базами данных (PostgreSQL) и современными веб-фреймворками.

## 2. Функциональные возможности

### Пользовательские роли
*   **Reader (Читатель):** Просмотр опубликованных статей.
*   **Author (Автор):** Создание черновиков статей, редактирование своих статей, просмотр статуса.
*   **Editor (Редактор):** Модерация статей, публикация черновиков.
*   **Admin (Администратор):** Полное управление контентом и пользователями.

### Функционал
*   Регистрация и аутентификация пользователей (JWT).
*   CRUD-операции (Create, Read, Update, Delete) для сущностей "Статья" и "Категория".
*   Система статусов статей (Черновик, Опубликовано, Архив).
*   Аудит действий в базе данных (логи изменений).

## 3. Стек технологий

**Frontend (Клиентская часть):**
*   React.js
*   TypeScript
*   React Router DOM
*   Axios (HTTP-клиент)
*   Vite (Сборщик)

**Backend (Серверная часть):**
*   Python 3.13
*   FastAPI (Веб-фреймворк)
*   SQLAlchemy (ORM, асинхронный режим)
*   Uvicorn (ASGI-сервер)
*   Pydantic (Валидация данных)
*   Python-Jose (JWT аутентификация)

**Database (База данных):**
*   PostgreSQL
*   PL/pgSQL (Хранимые процедуры, триггеры, функции)

## 4. Структура базы данных

Согласно требованиям курсовой работы, в базе данных реализованы следующие объекты:

*   **Таблицы:** Users, Articles, Categories, Article_Categories, Audit_Log.
*   **Триггеры:**
    *   Автоматическое обновление поля `updated_at` при изменении записи.
    *   Запрет на удаление опубликованных статей.
    *   Логирование создания новых статей в таблицу аудита.
*   **Хранимые процедуры и Функции:**
    *   Генерация уникальных URL-адресов (slug) для статей.
    *   Расчет времени чтения статьи.
    *   Поиск статей по ключевым словам.
*   **Представления (Views):**
    *   Список опубликованных статей с данными авторов.
    *   Статистика авторов.

Скрипты для создания структуры БД находятся в папке `DB/scripts/`. Полный бэкап базы данных содержится в файле `DB/BIVT-24-5_Martinson_DV_Newspaper.sql`.

## 5. Установка и запуск

### Предварительные требования
*   Установленные Node.js и npm.
*   Установленные Python 3.10+ и PostgreSQL.
*   Настроенное окружение `.env` (файл не включен в репозиторий).

### Запуск Backend

1.  Перейдите в директорию бэкенда:
    ```bash
    cd sourse/backend
    ```
2.  Создайте и активируйте виртуальное окружение:
    ```bash
    python -m venv venv
    # Для Windows
    venv\Scripts\activate
    ```
3.  Установите зависимости:
    ```bash
    pip install fastapi sqlalchemy psycopg2-binary python-jose passlib uvicorn
    ```
4.  Запустите сервер:
    ```bash
    uvicorn app.main:app --reload
    ```
    API будет доступен по адресу: `http://127.0.0.1:8000`

### Запуск Frontend

1.  Откройте новый терминал и перейдите в директорию фронтенда:
    ```bash
    cd sourse/frontend
    ```
2.  Установите зависимости:
    ```bash
    npm install
    ```
3.  Запустите сервер разработки:
    ```bash
    npm run dev
    ```
    Приложение будет доступно по адресу: `http://localhost:5173`
